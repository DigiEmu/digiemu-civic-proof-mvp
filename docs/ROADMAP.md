\# DigiEmu Civic Proof MVP – Roadmap



\## Ziel



In 4 Wochen entsteht ein kleiner, präsentierbarer Prototyp für Ursula:



\*\*Behörden-/CO₂-Vorgang → Entscheidung → Belegnummer → Audit → DigiEmu PASS/FAIL\*\*



\---



\## Woche 1 – MVP-Kern



\- Next.js Mini-App aufsetzen

\- Demo-Vorgang definieren

\- CO₂-Wert und Budget erfassen

\- Entscheidung erfassen

\- Belegnummer erzeugen

\- Snapshot anzeigen

\- Hash und Replay Hash erzeugen

\- PASS/FAIL anzeigen



Status: in Arbeit / teilweise umgesetzt



\---



\## Woche 2 – Präsentations-MVP



\- Audit Log anzeigen

\- Ampelstatus ergänzen

\- JSON Audit Report exportieren

\- Demo-Script für Ursula erstellen

\- MVP-Spezifikation schreiben

\- Screenshot vorbereiten



Status: in Arbeit



\---



\## Woche 3 – DigiEmu-Anbindung



\- DigiEmu Core API anbinden

\- Canonical JSON über Core prüfen

\- Hashing und Replay aus DigiEmu Core beziehen

\- lokale Browser-Demo durch echte Verifikationsschicht ersetzen



Status: geplant



\---



\## Woche 4 – Ursula-Demo



\- Demo-Daten finalisieren

\- Präsentationsablauf üben

\- kurzer Bericht erstellen

\- Feedback von Ursula aufnehmen

\- nächste Ausbaustufe definieren



Status: geplant



\---
